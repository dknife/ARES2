1. 개요

본 문서는 ARES 교육 플랫폼의 연결·통신·블록엔진·액추에이터·센서·설정·안정성·시뮬레이터·E2E 전 영역에 대한 테스트 수행 결과를 정리함. 테스트는 실제 UART 명령 프로토콜과 블록-명령 매핑에 대응되는 TCL v1.0 98 케이스를 기준으로 하였으며, 하드웨어 의존 항목은 실기(BLE)로, 그 외는 Web/Sim_Parts 시뮬레이터로 병행 검증함

7. 중점 점검 결과 (실제 소스 기준)

실제 코드 특성상 결함이 잘 드러나는 지점을 우선 점검한 결과는 다음과 같음

  • 부팅 None 가드 : 모듈 미장착 시에도 boot 완료·응답 정상 확인 (hardware.py try/except, main.py None 체크) - Pass
  • 응답 정책 동기화 : Web FIRE_AND_FORGET_HEADS 와 Pico NO_RESPONSE_CMDS 집합 일치 확인(28개 head 완전 일치, bracket 패턴 별도 규칙 포함) - Pass
  • 비상 정지 ABORT_CMDS : STOP_ALL/STOP이 SERVO_tFORWARD·DC_tFORWARD·SLEEP 등 진행 중인 타임드 동작도 즉시 중단시키는지 확인(TC-CONN-007) - Pass
  • GUN_FIRE 특이사항 : gun=None이어도 다른 핸들러의 None 가드 패턴과 달리 항상 1을 응답하는 것을 확인 - fire-and-forget이라 Web 동작에는 영향 없어 기능 결함 아님, 참고사항으로 기록
  • BATCH 금지 블록 : 값반환/제어흐름/중첩 차단 이중 방어 정상 - Pass
  • LED 6개·자기 디지털·연속 서보 : 하드웨어 사양이 코드와 일치(6개, active-low, 연속회전) - Pass
  • DC 모터 대칭 : dual PWM brake-modulated 전후진 출력 ±4% 근사 대칭 - Pass (정밀 실측 TC-DCM-002 보류)
  • 설정 영구 저장 : SYS_SET 후 재부팅해도 ModelFactorySetting.txt 값 유지 확인 - Pass
  • BLE 끊김 시 안전 중지 : 구동 중 GATT 끊김 시 실행 중단·재연결 안내 정상 (TC-BLE-007) - Pass
  • 배터리 전압 감시 : get_battery_voltage() 미구현 확인 - 개선 항목으로 이관 (BUG-002)

8. 미해결 이슈 및 우회

  • UART 9600 baud 처리량 천장 (BUG-001) : AT+BAUD 상향 + Pico UART_BAUDRATE 동기화 후속 과제 (STP 리스크 5)
  • 배터리 전압 감시 미구현 (BUG-002) : 전압 ADC 측정·경고 추가 후속 과제, 단기 완충 점검 매뉴얼화 (STP 리스크 6)
  • BATCH 누적 SLEEP > 5초 timeout 오인 : BATCH 안 SLEEP 누적 4초 이내 권장으로 운영 우회 (STP 리스크 4)
  • 보류 5건 : 오실로스코프·장시간·다수 기기 등 장비/일정 의존 항목으로, 현장 실증 단계에서 순차 수행 예정
  • 시뮬레이터 자기 센서 미구현 : Web/Sim_Parts의 MAGNET 응답은 항상 MAG:0으로 고정되어 자기 감지 시나리오는 시뮬레이터로 검증 불가 - TC-SEN-003 실기 테스트로만 커버, 시뮬레이터 자기 감지 구현은 후속 과제로 등록
