# MISSIONS.md — ARES 12차시 × 4미션 교육과정 요약

본 문서는 `Missions/` 폴더의 LaTeX 워크북(`main.tex` + `chapter01~12.tex`)을
요약한 자료이다. ARES 블록 코딩 학습 플랫폼의 **1분기 12차시 × 4미션 =
총 48미션** 커리큘럼을 한눈에 확인할 수 있게 정리하였으며, 향후 웹 UI
(`Web/main.html` 블록 에디터 / `Web/index2.html` 미션 페이지)와 펌웨어
(`Pico/process_data.py` 명령 디스패처) 개발 시 참조 자료로 활용한다.

> 원본: `Missions/main.tex` (XeLaTeX, minted, 한글) — 컴파일 결과 PDF는
> `Missions/main.pdf`. 한글 주석 처리 정책은 `Missions/COMMENTPOLICY.md`
> (minted 필수, listings 금지) 참조.

## 1. 커리큘럼 한눈에 보기

| 차시 | 주제 | 핵심 하드웨어 | 핵심 개념 | 태그 |
|-----|------|------------|----------|------|
| 1   | 코딩 입문 & 알비 만남 | (이론) Bluetooth 페어링 | 순차/반복 개념, 앱 설치 | `theory` |

| 5   | LED 3개 신호등 | LED 3개 | 시퀀스 사고, 모드 분기 (주간/야간/긴급) | `TRAFFIC` |

| 10  | LED 5개 시퀀스 & 카운트다운 | LED 5개 | 발사 준비/카운트다운/파티, 모듈화 사고 | `SEQUENCE` |

| 12  | 화성 로켓 최종 발사! | LED 5개 + 부저 + DC모터 | 통합 시나리오, 자유 창작 발표 | `LAUNCH` |

### 교육과정 설계 흐름

```
[1차시 이론]  → [2~4차시 단일 출력 입문 (LED→부저)]
              → [5~6차시 다채널/난수]
              → [7~8차시 운동(DC모터)]
              → [9차시 회고/제작 휴식]
              → [10~12차시 통합 프로젝트 "로켓 발사"]
```

### 3차시 — LED 2개로 표정 만들기 (`WINK`)

다채널 동시 제어로 확장. 알비의 "눈빛" 감정 표현.

| # | 미션명 | 학습 목표 요약 |
|---|--------|--------------|
| 1 | 알비의 눈이 반짝! | LED 1·2 동시 점등, `while True` 무한 깜빡 |
| 2 | 눈 번갈아 깜빡이기 | 좌→우 교대(0.5초), 5회 반복 |
| 3 | 알비가 깜짝 놀랐어요! | 0.1초 간격 빠른 10회 + 0.5초 휴지 (놀람) |
| 4 | 윙크 리듬! | 좌 2회 → 1초 휴식 → 우 2회 패턴 5회 |

**대표 코드(미션1):**
```python
while True:
    led_on(1, 1.0); led_on(2, 1.0)
    time.sleep(1)
    led_off(1); led_off(2)
    time.sleep(1)
```

**창작 활동:** 미션 4에서 학습자가 "화남(빨강+빠름)", "졸림(점점 느림)"
등 자신만의 감정 표현 자유 설계.

### 4차시 — 부저로 소리 만들기 (`BUZZER`)

소리라는 새 출력 모달리티. **주파수(Hz)** + **지속시간(s)** 두 매개변수.

| # | 미션명 | 학습 목표 요약 |
|---|--------|--------------|
| 1 | 부저 기초 삐빅 3번 | `buzzer_on(500, 0.5)` 3회 반복 |
| 2 | 화성 앰뷸런스 사이렌! | 400Hz ↔ 800Hz 교대 무한 |
| 3 | 운석 충돌 긴급 경보! | 0.1초 간격 10회 + 0.5초 휴지 |
| 4 | 긴급 구조 SOS 신호! | 모스 부호 ···───··· |

**대표 코드(미션4 SOS):**
```python
while True:
    for i in range(3): buzzer_on(800, 0.2); time.sleep(0.1)   # ···
    time.sleep(0.3)
    for i in range(3): buzzer_on(800, 0.6); time.sleep(0.1)   # ───
    time.sleep(0.3)
    for i in range(3): buzzer_on(800, 0.2); time.sleep(0.1)   # ···
    time.sleep(1)
```

### 6차시 — 랜덤 함수와 가위바위보 게임 (`RANDOM`)

`import random`, `random.randint()` 도입. 비결정성 학습.

| # | 미션명 | 학습 목표 요약 |
|---|--------|--------------|
| 1 | 랜덤 가위바위보 시작! | `random.randint(1,3)` → 해당 LED 점등 |
| 2 | 하나 빼기! | 시작 깜빡 + `one_out()` 블록 (2개 켜고 1개 끄기) |
| 3 | 알비를 이겨라! | `random_led()` 5회, 학습자가 이기는 손을 빠르게 |
| 4 | 가위바위보 왕중왕! | 3라운드 + 챔피언 축하 댄스 |

**대표 코드(미션1):**

import random
choice = random.randint(1, 3)

### 7차시 — DC모터 입문: 회전과 룰렛 (`MOTOR`)

운동(움직임)을 코드로 제어하는 첫 단계. PWM 도입.

| # | 미션명 | 학습 목표 요약 |
|---|--------|--------------|
| 1 | DC모터 기본 회전 | `motor_forward(3)` / `motor_backward(3)` / `motor_stop()` |
| 2 | 속도 조절 빠르게 느리게! | `motor_forward(2, 100)`, `motor_forward(2, 50)` — PWM 듀티 % |
| 3 | 차림표 원판 룰렛! | 100→70→40→0% 서서히 감속 |
| 4 | 나만의 화성 룰렛 게임! | 전진 감속 후 미세 후진으로 정확 정지 |

**API 패턴:** `motor_forward(seconds [, percent])`, `motor_backward(seconds [, percent])`, `motor_stop()`.

**주의:** DC모터는 큰 전류 → USB 전원으로 부족 가능, 별도 배터리 권장.
정지 시 반드시 `motor_stop()` 호출 (전류 차단 명시).

### 8차시 — 알비 카트 주행 (`MOTOR`)

7차시 모터 + 바퀴 결합 → "움직이는 로봇" 완성.

| # | 미션명 | 학습 목표 요약 |
|---|--------|--------------|
| 1 | 알비 카트 앞으로! | `motor_forward(3)` 단순 전진 |
| 2 | 알비 카트 뒤로! 왕복 | 전진(3s) → 정지(1s) → 후진(3s) |
| 3 | 속도 조절 주행! | `motor_forward_pwm(30/60/100)` 가속 + 감속 |
| 4 | 알비 카트 나만의 코스! | 전진·후진 반복 자유 조합 |

**새 API:** `motor_forward_pwm(percent)` — 시간 인자 없이 듀티만 설정 후
연속 운행. 8차시에서 처음 등장.

### 10차시 — LED 5개 시퀀스 & 카운트다운 (`SEQUENCE`)

LED 5개로 확장. 발사 시퀀스 구현. **모듈화 사고**의 시작.

| # | 미션명 | 학습 목표 요약 |
|---|--------|--------------|
| 1 | 발사 준비 순차 점등! | LED1~5 순차 점등 + 5개 동시 깜빡 3회 |
| 2 | 발사 파티 조명! | 동시 깜빡 + 파도 패턴(1→2→3→4→5→4→3→2→1) |
| 3 | LED 카운트다운 5→1! | 5개 켜고 1초마다 하나씩 끔, 끝나면 발사 깜빡 |
| 4 | 나만의 발사 조명쇼! | 준비→카운트다운→발사→파티 통합 |

**대표 코드(미션3 카운트다운):**
```python
for i in range(1,6): led_on(i,1.0)
for i in range(5, 0, -1):
    led_off(i); time.sleep(1)
```

**핵심 포인트:** 미션 4는 "서브시퀀스의 합성"을 명시적으로 가르치는 첫
미션 — 앞선 3개 미션 코드를 복사·이어붙여 통합 작품 완성.

### 11차시 — LED + 부저 동기화 (`SYNC`)

빛과 소리 두 채널 동시 제어. 음계(도레미파솔) 등장.

| # | 미션명 | 학습 목표 요약 |
|---|--------|--------------|
| 1 | LED + 부저 동시 제어! | `led_on` + `buzzer_on` 동시 5회 |
| 2 | 카운트다운 발사 시스템! | LED 끄기 + 부저 400→800Hz 점진 + 발사음 1200Hz |
| 3 | 발사 성공 세레머니! | LED 파티 + 도레미파솔(262/294/330/349/392Hz) |
| 4 | 나만의 발사 사운드트랙! | 준비→카운트다운→발사→축하 4단계 자유 작곡 |

**음악 노트:** 도레미파솔 = C4 옥타브(262, 294, 330, 349, 392Hz). 부저가
"단순 경보"가 아니라 "작곡 도구"가 될 수 있음을 발견.

### 12차시 — 화성 로켓 최종 발사! (`LAUNCH`)

1분기 대단원. **LED 5개 + 부저 + DC모터** 통합 시나리오.

| # | 미션명 | 학습 목표 요약 |
|---|--------|--------------|
| 1 | 발사 전 최종 점검! | LED→부저→모터 순차 점검 + 완료 신호 |
| 2 | 로켓 발사 카운트다운! | 경고음 3회 + LED 5→1 + 음 점차 상승 |
| 3 | 화성 로켓 발사!!! | 사이렌 + 모터 + LED 점화 + 폭죽 깜빡 |
| 4 | 1분기 자유 창작 발표! | 모든 부품 자유 조합, 친구들 앞에서 시연 |

**대표 코드(미션3 최종 발사):**
```python
buzzer_on(400, 0.5); buzzer_on(800, 0.5)   # 사이렌
motor_forward(2)                           # 발사 장치
for i in range(1,6): led_on(i,1.0); time.sleep(0.1)
for j in range(8):
    for i in range(1,6): led_off(i); time.sleep(0.1)
    for i in range(1,6): led_on(i,1.0); time.sleep(0.1)
```

**칭호:** 미션 4 완료 학습자는 "★ 화성 탐사 훈련생 ★" 칭호 수여.

## 3. 1분기 누적 요약

### 배운 부품 (누적)

| 차시 | LED | 부저 | DC모터 | 추가 부품 |
|------|-----|------|--------|----------|
| 2    | 1개 | -    | -      | - |
| 3    | 2개 | -    | -      | - |
| 4    | -   | ★    | -      | - |
| 5~6  | 3개 | -    | -      | - |
| 7    | -   | -    | ★      | 원판 |
| 8    | -   | -    | ★      | 바퀴 2개 |
| 10   | 5개 | -    | -      | 발사대 건물 |
| 11   | 5개 | ★    | -      | - |
| 12   | 5개 | ★    | ★      | 발사용 건 |

### 배운 개념

디지털 출력(HIGH/LOW) → 시간 지연 → 순차 → 반복(for / while) →
다채널 동시 제어 → 주파수 제어 → 모드 분기 → 난수 → PWM 듀티 사이클 →
다채널 동기화 → 시나리오 통합·모듈화

### 대표 작품

알비 눈빛(2차시) · 화성 신호등(5차시) · 가위바위보 게임(6차시) ·
알비 카트(8차시) · 로켓 발사 시스템(10~12차시).

## 4. 웹 서비스 개발 시 참고 사항

### 4.1 Blockly 블록 카탈로그 (커리큘럼 기반)

각 차시에서 학습자가 사용할 수 있어야 하는 블록:

| 차시 시작 | 신규 블록 / 함수 | 비고 |
|---------|-----------------|------|
| 2       | `led_on(num, brightness)`, `led_off(num)`, `time.sleep(s)` | LED 1~5번 |
| 3       | (동일) — 다채널 사용법 | - |
| 4       | `buzzer_on(freq, duration)` | freq는 정수 Hz |
| 5       | (LED 3개 사용) | - |
| 6       | `random.randint(a,b)`, `random_led()`, `one_out()` | 매크로 블록 |
| 7       | `motor_forward(s [, %])`, `motor_backward(s [, %])`, `motor_stop()` | PWM 인자 옵션 |
| 8       | `motor_forward_pwm(%)` | 시간 인자 없는 변종 |
| 10      | LED 5개 (`led_on(1)` ~ `led_on(5)`) | - |
| 12      | (통합) — 모든 블록 동시 사용 | - |

> 위 함수명은 워크북 샘플 코드의 추상 표현이다. 실제 펌웨어 명령 매핑은
> `Document/API_DOCUMENTATION.md` 및 `Pico/process_data.py` 참조 — 예:
> `LED_ON,num,brightness`, `tFORWARD,N`, `MAIN_FORWARD` 등.

### 4.2 미션 페이지 (`Web/index2.html`) 구조 제안

각 미션 페이지는 워크북의 4요소를 그대로 반영하면 자연스럽다:

```
┌─────────────────────────────────────────┐
│ [차시] X차시 — 부제                       │
│ [미션] 미션 N: 미션명          [태그]    │
├─────────────────────────────────────────┤
│ 💬 스토리박스 (알비 ↔ 아레스 대화)        │
├─────────────────────────────────────────┤
│ 🎯 학습 목표 (체크리스트)                 │
├─────────────────────────────────────────┤
│ 🧩 [Blockly 작업 영역]                   │
│   - 미션별 토글된 블록만 노출             │
├─────────────────────────────────────────┤
│ 💡 샘플 코드 보기 / 힌트                  │
└─────────────────────────────────────────┘
```

### 4.3 진행 상태 관리

- **차시 잠금 해제**: 1차시 → 12차시 순차 진행. 이전 차시 완료 시 다음
  차시 해제.
- **미션 완료 판정**: BLE 응답 또는 학습자 자가 체크.
- **이론 차시(1·9)**: 코드 실행 없이도 "읽음" 버튼으로 완료 가능.

### 4.4 컴파일·문서 운영

워크북 PDF 갱신 시: `Missions/` 디렉터리에서
```bash
xelatex -shell-escape main.tex
xelatex -shell-escape main.tex   # 목차 수렴
```
`-shell-escape` 누락 시 minted 컴파일 실패. 자세한 사항은
`Missions/COMMENTPOLICY.md` 참조.

## 5. 향후 확장 (2분기 예고)

워크북 9차시·12차시에 명시된 2분기 학습 방향:

- **서보모터** — 정밀 각도 제어
- **초음파 센서** — 거리 측정 입력
- **LCD** — 텍스트 출력

→ 학습 모델이 "출력 중심"에서 "**입력 + 정밀 제어**"로 확장됨.
