# SIMULATOR.md — 시뮬레이터 동작 설계 및 구현 가이드


---

## 1. 시뮬레이터의 용도

- **키워드**: 장치 연결이 되어 있지 않아도 블록 코딩 테스트를 할 수 있는 기능
- **주요 기능**: 블록코딩 창의 코드를 시뮬레이터가 가진 각종 씬(Scene)에 전달하여 동작시킴
- **씬 구성**: WebGL 렌더링이 가능한 glb 파일을 로딩하거나, 박스, 구(sphere), oled 표현을 위한 직사각형 등의 객체를 로딩하고, 전체 씬을 저장할 수 있음
- **사용자**: 사용자는 씬의 구성을 하지 못하고, 개발자 모드에서만 씬을 생성하고 저장할 수 있음. 사용자는 저장된 씬만 읽을 수 있음

## 2. 씬의 생성, 저장, 로딩

### 씬의 생성
- 씬 선택 메뉴에 "새로운 씬 만들기"를 통해 빈 씬을 생성
- 현재 씬의 객체들 하이어라키(hierarchy)를 보여주는 인터페이스를 유지
- 하이어라키는 Scene을 루트로 하고, 객체를 담아 나가게 함. 최초에 루트가 선택된 상태이고, 담긴 객체를 개별적으로 선택할 수 있음.
- 특정한 객체를 선택한 상태에서 객체를 추가하면 선택된 객체의 하위에 객체가 생성됨
- 특정 객체를 선택하고 삭제를 하면 해당 객체와 하위 객체가 모두 삭제됨
- 객체는 특정한 컴포넌트를 가질 수 있게 하고, 해당 컴포넌트의 특성에 따라 블록 코딩 명령에 반응함.
- 생성한 씬은 저장할 수 있도록 저장 메뉴가 있음.

## 3. 구현 규약 (2026-07-08 확정)

- **이동·회전 적용 주체** — 씬 구성자가 결정한다. 컴포넌트는 자신이 부착된 객체(+하위 객체)만
  이동·회전시키고, **설정된 필드(축)만** 동작한다. 예: 바퀴가 돌면서 로버 전체가 움직이는 구성이라면
  이동 축(`axis_direction`/`axis_translate`)은 로버 루트 객체에, 바퀴 회전축(`axis_rotation`)은
  각 바퀴 객체에 부착한다. 시뮬레이터는 주어진 조건대로 하위 객체만 데리고 이동·회전하면 된다.
- **좌표계** *(2026-07-09 재확정)* —
  - **회전축**(`axis_rotation`)·**선회축**(`axis_turn`)의 **방향**은 **부모 객체 좌표계**
    기준이다. 객체 자신이 어떤 자세로 돌아도 축 방향은 부모 기준으로 고정된다.
  - 각 **기준점 오프셋**(`rotation_offset`·`turn_offset`)은 **해당 객체의 로컬 좌표계**
    기준이다. 로컬로 정의된 기준점은 객체에 붙은 재질점이라 객체가 어떤 변환 상태
    (이동·회전)에 있어도 항상 같은 자리에 놓이고, 회전 중에도 축이 공간에 고정된 채
    객체가 그 둘레를 돈다. 거리는 m 그대로(스케일 미적용).
  - **이동 방향 축**(`axis_direction`·`axis_translate`)은 **해당 객체의 로컬 좌표계**
    기준이며 **오프셋이 없다** — 선회 후 전진하면 틀어진 몸체 방향으로 이동한다.
  - 센서·연출 점(UltraSonic ray 방향·Magnet 감지점·Gun 발사/연기점)은 객체 로컬 기준.
  *(변경 이력: 최초 월드 좌표계 → 회전 후 원래 축으로 이동하는 문제로 로컬로 개정 →
  회전 관련 축·오프셋은 부모 좌표계로 재확정 → 기준점 오프셋은 변환 상태와 무관하게
  같은 자리에 놓이도록 **로컬 좌표**로 최종 확정, 축 방향만 부모 좌표계.)*
- **거리 단위** — 씬 좌표 **1 unit = 1 m**. 전역 단위 배율(`unitScale`)을 씬 데이터에 두어
  추후 단위 조정이 가능하게 한다.
- **Magnet 감지 반경** — `detection_point` 기준 **5 cm(0.05 unit)** 이내에 Metal 객체가 있으면 1, 없으면 0.
- **발사대 재현** — 로켓 발사 명령은 **소리만** 재생한다(별도 Rocket 컴포넌트 없음).
  레이더 회전은 **DC 컴포넌트의 회전(`axis_rotation`)** 으로 구성한다.
- **개발자 모드 진입** — **Ctrl+E** 토글. 개발자 모드에서만 씬 생성·편집·저장 UI를 노출하고,
  사용자 모드에서는 저장된 씬 로드만 가능하다.
- **객체 복제** *(2026-07-09 추가)* — 개발자 모드에서 객체 선택 후 **Ctrl+V**
  (Ctrl+D 는 브라우저 북마크 단축키와 충돌해 V 사용). 원본과 같은
  부모를 갖는 **sibling** 으로 복제되며, 라벨은 `원본이름_dup`, 위치는 부모 좌표계 기준
  **x축 +1 m**. 하위 객체와 부착 컴포넌트도 함께 복제된다.
- **박스·구 색상** *(2026-07-09 추가)* — 박스와 구는 **기본색**과 **발광색**을 각각
  `[r,g,b,a]`(0~1)로 갖는다(`a`=불투명도). 평상시는 **기본색으로만** 렌더링되고,
  LED 컴포넌트가 부착된 객체는 **LED 밝기 t(0~1)에 따라 기본색↔발광색이 보간**된다 —
  t=0 이면 기본색, t=1 이면 발광색만 보이며, 불투명도(a)도 함께 보간된다.
  생성 시 디폴트(박스 주황·구 하늘색, 발광색 흰색)로 만들어지고 인스펙터의 색상 행에서
  편집한다. 값은 `metadata.colors` 로 저장되어 씬 직렬화(`colors` 필드)·복제에 함께 실린다.
- **회전축 시각 편집** *(2026-07-09 추가)* — 회전 특성 컴포넌트(DC 회전축, Servo
  스핀축·선회축)를 가진 객체를 선택하면 **시뮬레이션 화면 하단 중앙**에 축 버튼
  (회전축/스핀축/선회축)이 나타난다. 버튼을 켜면 축이 지나는 지점에 핸들(노란 구 + 축 라인)이 표시되고, 이동
  기즈모로 핸들을 끌어 놓은 위치가 그대로 **회전기준(`rotation_offset`)·선회기준
  (`turn_offset`)** 값이 된다(**객체 로컬 좌표**로 저장 — 어떤 변환 상태든 같은 자리).
  핸들을 객체 원점에 두면 오프셋 미사용으로 정리된다.
- **개발 중 씬 임시 보존** *(2026-07-09 추가)* — 개발자 모드에서 작업하던 씬은 주제
  전환·모드 이탈·새로고침 시 localStorage 에 자동 임시 저장된다(기반 주제 포함).
  개발자 모드 재진입(빈 화면일 때)이나 DEV 바 **새 씬**에서 "이어서 작업할까요?"로
  복원을 제안하며, **씬 저장/서비스 등록이 완료되거나 새 씬 시작을 확정하면** 임시본을
  폐기한다. 개발자 '빈 씬' 작업 중 시뮬레이션 화면을 닫았다 열어도 씬은 유지된다.
- **서비스 등록/제거** *(2026-07-09 추가)* — 개발자 모드 DEV 바의 **서비스 등록**은 현재
  화면(기반 주제 + 스폰 객체)을 `Web/scenes/<id>.json` 으로 직렬화하고
  `scenes/manifest.json` 의 `scenes` 목록에 추가해, 서비스 제공 주제와 **같은 형식**으로
  주제 드롭다운에 노출한다. **서비스 제거**는 선택된 주제를 서비스에서 뺀다 — 등록 주제는
  `scenes` 에서 제거(씬 파일은 보존), 기본 주제(알비 등)는 manifest 의 `hiddenTopics` 에
  넣어 숨긴다. 파일 기록은 File System Access API 로 `Web/scenes` 폴더에 직접 쓴다 —
  최초 1회만 폴더를 선택하면 핸들이 IndexedDB 에 저장되어 이후 등록/제거는 선택 창 없이
  자동 기록된다(상위 폴더를 골라도 `Web/scenes` 로 자동 진입). 로컬 서비스에 즉시
  반영되고 git push 로 배포에도 반영되며, 미지원 브라우저는 갱신 파일 다운로드로
  폴백한다. 저장된 씬은 로드 시 `topic` 필드의 기반 주제 위에 재구성된다.
- **회전 기준축 오프셋** *(2026-07-08 추가)* — 회전 컴포넌트의 각 회전축에 선택 필드로
  기준점 오프셋(로컬 좌표의 점)을 줄 수 있다. 회전축이 객체 원점이 아니라 그 점을
  지나도록 옮겨진다. DC: `rotation_offset`(회전축), Servo: `rotation_offset`(스핀축)·
  `turn_offset`(선회축). 예: 문(경첩) 회전, 바퀴 메시 원점이 축과 어긋난 모델 보정.
- **스케일 비전파** *(2026-07-08 확정)* — 스케일은 **개별 객체에 한정**되고 하위 객체로
  전달되지 않는다. 상위 노드를 스케일해도 하위 노드의 모양이 변형되지 않으며, 하위
  객체의 위치 오프셋 거리(m)도 그대로 유지된다. (구현: 하위 simObject 는 부모의
  역스케일 자식 홀더에 부착)
- **점 오프셋의 의미** *(2026-07-09 최종)* — 회전 기준점 오프셋은 **객체 로컬 좌표의
  점**으로, 회전축을 그 점을 지나도록 옮겨 배치한다는 뜻이다. 예: 오프셋 `(0,1,0)` 이면
  객체 로컬 y 방향 1 m 지점을 축이 지나고, 객체는 그 **공간에 고정된 축** 둘레를 돈다
  (문 경첩, 원점이 어긋난 바퀴 보정). 로컬 좌표라 객체가 어떤 변환 상태(이동·회전)에
  있어도 기준점은 항상 객체의 같은 자리에 붙어 있고, 회전이 진행돼도 축이 따라 움직이지
  않는다. (감지점·연기점도 객체 로컬 기준.)
  **거리에 객체 스케일을 적용하지 않는다** — `(0,1,0)` 이면 스케일과 무관하게 정확히
  1 m 위를 축이 지난다.
