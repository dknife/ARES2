## 2026-07-02 — 서비스 UX 개편 (개요 통합 · 하단바 · 애니메이션)

### 개발 문서
- `dev.html`이 이제 **3부 구성**: ① 1차 Refactoring 계획 ② 2차 Refactoring 계획 ③ 디자인 계획.
  상단에 파트 점프 내비 존재. 원문은 `Document/REFACTORING_PLAN*.md`, `DESIGN.md`.

### 서비스 첫 화면 (`Web/index.html`)
- 인사 문구 축약: `안녕! 나는 탐사로봇 알비야 👋` + `이제 탐험을 시작해!` (이전 장문 안내 제거).

### 개요 화면 = 단일 통합 아코디언 (`Web/overview.html`, `main.js`, `styles.css`)
- 기존 개요·차시·미션 3단계를 **개요 안 중첩 아코디언**으로 통합.
  - 차시 클릭 → (위) **차시 정보**(푸른색 팔레트) + (아래) **미션 리스트**.
  - 미션 클릭 → 그 아래로 **미션 상세**(이야기·목표·샘플코드) + `🧩 미션 코딩 시작` 버튼.
  - `미션 코딩 시작` → 해당 미션 선택 후 **블록코딩 모드**로 진입.
- 관련 헬퍼: `renderInlineMissionItem/Detail`, `renderInlineLessonInfo`, `toggleInlineMissionDetail`, `openMissionCoding`.
- 색 구분: **차시 정보 = 파랑 계열**, **미션 = 주황 계열**.

### 하단 모바일 내비게이션 (`Web/main.html`, `main.js`, `styles.css`)
- 최종 구성 4탭: **🚀 미션 · 🧩 코딩 · 🤖 AI · 📝 로그** (그리드 `repeat(4, …)`).
  - 이전 개요·차시·미션 3탭 → `미션` 단일 탭으로 통합했다가, 이후 `코딩` 탭 추가.
- **미션 탭** = 개요 메뉴(아코디언) 표시. **코딩 탭** = 기록된 미션의 블록코딩으로 바로 진입.
- 활성 표시: 미션(코딩 영역) 뷰 → `코딩` 탭, 개요·차시 메뉴 → `미션` 탭.

### 선택 미션 기록 (localStorage 키)
- `ares_last_coding_mission_v1` → `{ lesson, mission }`. `enterMission()`에서 기록(`rememberCodingMission`).
- `코딩` 탭과 `AI` 탭이 이 값을 공유해, **1차시 1미션으로 리셋되지 않고 마지막 미션 유지**.
- (참고) 미션 완료 진행도 키: `ares_completed_missions_v1`.

### 애니메이션 (`styles.css` + `main.js`)
- **아코디언 여닫이**: 열림 `2s` / 닫힘 `1s`. 실제 콘텐츠 높이를 `--acc-h`로 측정해 자연스럽게.
  제어는 `openAccordion()/closeAccordion()` (클래스 `accordion-opening/closing`).
- **화면 전환**: 개요/차시/미션 뷰 전환 = 슬라이드-인, 미션 내부 모드(설명↔코딩↔시뮬) = 페이드. 둘 다 `1s`.
- `prefers-reduced-motion` 사용자는 애니메이션 생략.

### 모바일 캐시 이슈 (미해결 — 다음 작업 후보)
- 증상: PC는 최신인데 **모바일은 이전 화면**. 원인은 서비스워커 아님(없음) → **브라우저 HTTP 캐시**.
- 핵심: `main.html`이 `styles.css`/`main.js`를 **버전 없이** 참조 → 모바일이 옛 JS/CSS 유지.
- **주의(중요)**: `main.js`는 `type="module"`로 8개 모듈을 import. **엔트리 스크립트에 `?v=`를 붙여도
  내부 `import`문에는 전파되지 않음** → 하위 모듈만 고치면 반영 안 됨.
  → 날짜 쿼리를 엔트리에만 붙이는 방식은 부적합. **import map** 또는 **빌드 시 버전 자동 주입** 권장(미적용).
- 임시 우회: 시크릿 탭 / 캐시 삭제 / URL `?v=` 쿼리.
