TRACK 1

시뮬레이터

1.1요소별 코드 분할

Web/simulation.js(약 2,333줄)의 거대 클로저를 요소 단위(로버 부품·발사대·신호등·LED·총·로켓·음파·OLED)로 분할한다. 1차 계획 유형 A의 sim/ 서브시스템 구조(context·assets·leds·movement·gun·rocket·traffic·waves·oled·render·audio·dispatch·topics)를 실제로 도입해, 한 요소를 독립적으로 보고 고칠 수 있게 한다.

↳ 1차 로드맵 6단계(simulation.js → sim/)와 동일 작업을 2차에서 본격 착수한다.

1.2단위 메쉬 로딩 이후 개발자 도구로 Transform 조작

부품 메쉬(Web/Mesh/RoverParts/*.glb, EnvAssets/*)를 개별(단위) 로딩한 뒤, 브라우저 개발자 도구로 위치·회전·스케일(Transform)을 실시간 조작해 정렬값을 잡고, 확정값을 코드(상수/설정)로 역반영한다. 하드코딩 좌표를 눈대중으로 맞추던 수작업을 도구화한다.

  • 구현 아이디어 — 디버그 모드(?debug=1)에서 three.js TransformControls 부착 → 선택 객체의 position/rotation/scale을 패널·콘솔에 출력 → 코드로 붙여넣기.
  • 단위 메쉬 로더 — 부품을 하나씩 로딩·점검하는 경로로 모델 교체·정렬 검증을 빠르게.
TRACK 2

피코 + 블록코딩

2.1요소 피코 코드 점검

Pico/process_data.py 핸들러와 하드웨어 모듈(모터·LED·부저·거리·자기·총·OLED)을 요소별로 점검·정리한다(1차 유형 B 분할과 연계). 각 명령 문자열이 실제 하드웨어 동작과 1:1로 맞는지 검증하고, 죽은 코드·중복 분기를 솎아낸다.

2.2미션별 블록 노출 시스템 개선

현재 toolbox는 모든 블록을 항상 노출한다(main.html 정적 XML). 미션(주제)에 따라 필요한 블록만 보여주는 노출 제어 시스템을 도입한다. 정적 구성을 미션 메타데이터 기반 동적 구성으로 전환한다(Missions/, XML Presets/, MISSIONS.md 연계).

2.3미션 주제별 필요 블록 설계

미션 맥락에 맞는 블록 세트를 설계한다. 같은 "전진"이라도 맥락에 따라 구분한다:

맥락블록핵심 파라미터
알비 DC 전진DC 모터 주행방향(전/후)
속도 조절주행 속도 설정speed 값
발사 동작로켓 발사 시퀀스카운트·점화

→ 맥락에 따라 블록의 명칭·파라미터·노출을 다르게 설계해, 학습자가 미션 목표에 집중하도록 한다.

TRACK 3

웹 인터페이스

3.1반응형 웹

데스크톱·태블릿·모바일 폭에 맞춰 레이아웃을 재구성한다. 현재 index.css는 @media 일부만 대응 → 체계적 브레이크포인트로 정리한다.

3.2모바일 환경 최적화 — 최소 요구 정보 추출

모바일에서는 블록 코딩·기기 연결 등 핵심 동작에 필요한 최소 정보만 노출한다. mobile-preview.js의 ?mobile=true 미리보기를 실제 모바일 UX로 발전시킨다(불필요한 패널 축소, 터치 타깃 확대).

3.3디자인 현대화

타이포·여백·컬러·컴포넌트를 현대적 디자인 시스템으로 정리한다. 홈 3D 히어로(우주 유영 연출)와 톤을 통일한다.
