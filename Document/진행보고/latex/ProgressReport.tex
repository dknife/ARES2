\begin{abstract}
\noindent
본 문서는 ARES(Autonomous Rover Exploration System) 교육용 로봇 플랫폼의
제품개발학기 현장 진행을 1$\sim$13일차에 걸쳐 기록한 보고서이다. ARES는
초등학생을 대상으로 한 블록 코딩 교육 플랫폼으로~\cite{papert1980, maloney2010scratch}, Google Blockly 기반 웹
애플리케이션~\cite{blockly}, 라즈베리파이 피코 MicroPython 펌웨어~\cite{micropython,rp2040},
그리고 보조 AI 모듈~\cite{exaone35}의 세 런타임 도메인으로 구성된다. 보고 기간 동안
컴퓨터실 개발 착수, 교구 제작 협력 업체(코리아사이언스) 방문과 웹 서비스 시연, 디자인 프로토타입
제작, 1차시 운영 점검 및 인터페이스 개선, \textbf{팀원별 작업 브랜치의
\code{main} 통합 머지와 통합 후 전 계층 서비스 점검$\cdot$트랙별(웹$\cdot$피코$\cdot$시뮬레이션)
개선}, UI 디자인 시안$\cdot$실서비스 적용과 교구 시제품 산업체 공동 점검, 그리고
\textbf{시뮬레이터 씬 생성 도구(컴포넌트 시스템) 구축과 QA 체계 기반 소프트웨어 테스트
본격 수행}을 진행하였다. 주요 산출물은 본문 「산출물」에 요약하고, \textbf{전문은
참고문헌 뒤 부록 A$\sim$G에 수록}한다 --- 부록 A$\sim$C(기준선 분석$\cdot$개선 설계),
\textbf{부록 D$\cdot$E}(통합 머지 중간 점검$\cdot$재분석), \textbf{부록 F}(소프트웨어 테스트
결과 보고서 STR v1.1), \textbf{부록 G}(시뮬레이터 사용법).
\end{abstract}
